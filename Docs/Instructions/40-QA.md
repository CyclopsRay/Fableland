# Fableland — QA / Testing Department

How we keep bugs out despite (a) one developer, (b) a dev host that often has **no
Godot/.NET toolchain**, and (c) a game whose core loop spans three layers. The answer
is a pyramid: cheap always-on checks at the bottom, expensive human playtests at the top.

---

## 1. The testing pyramid

```
        (5) Manual playtest protocols        — per milestone, seeded, checklisted
        (4) In-game debug harnesses          — seed field, Rest, Mist, tuning overlay,
                                               ShowDebugRanges, map view toggles
        (3) Pure-logic unit tests            — headless dotnet test on Scripts/Data + Map
        (2) Real build                       — dotnet build / Godot --headless (when available)
        (1) Static checks                    — every commit, no exceptions
```

### (1) Static checks — every commit
- Brace/paren balance in changed `.cs`; `.tscn` `ext_resource`/script paths resolve;
  `GetNode`/NodePath strings match the scene trees; exported-property assignments in
  scenes still point at existing members; no shadowed inherited members.
- Version trio in sync (`GameVersion.cs` / `VERSION` / Hud `VersionLabel`).
- Grep-audits worth running when touching the relevant systems:
  - `GD.Rand|new Random(` outside `DetRandom.NewSeed` → determinism leak.
  - Numeric literals in new gameplay logic → should be a table lookup (30-DATA §2).
- **Static checks miss type/API-existence errors** (Godot API surface!). Say so in the
  commit message when no real build ran.

### (2) Real build — whenever a toolchain is present
`dotnet build` at minimum; opening the Godot editor re-imports assets and surfaces
scene errors. If a session has a toolchain, spend the first minutes building HEAD —
catching a stale API break early beats debugging it inside a feature.

### (3) Pure-logic unit tests
The reason `Scripts/Data/` and map logic stay free of Godot types. Highest-value tests:
- **Map determinism:** same seed ⇒ byte-identical graph (serialize node ids/positions/
  edges → hash). Two different seeds ⇒ different hash. This single test guards the
  whole seed contract.
- **Map invariants (any seed, fuzz 100+ seeds):** the outer coast is one connected landmass with
  no detached outer islets; the static pentagonal VOID is within its target 1/30 area tolerance;
  five non-crossing, variable-width rivers each touch the VOID and coast, producing exactly five
  realm polygons; every realm has equal level-1/level-2 city totals with A/B sublevels and ranks
  LV4, both LV3s, `2-B`, `2-A`, `1-B`, `1-A` by VOID distance; every realm node reaches its `4-1` without a road
  entering another realm, a river, the VOID, or another road except at a shared node; exactly one
  LV3→LV4 approach per realm contains a Transportation Hub; each realm has 3–5 hubs and 4–6
  degree-two Event nodes; terrain is one of
  `low-ground`/`sea-level`/`high-ground`; TwistedReality only unlocks after the starting LV4,
  builds from a completed VOID-river peripheral city to the nearest city of the closest realm
  (or nearest LV5 from LV4), creates a midpoint Eidolon Shelter, rejects already-anchored
  endpoints, and observes its 4-day cooldown; zone-6 edges remain invisible; each outer combat node has a valid controlled field;
  function nodes devour only after a strict majority of their connected cities, and a devoured
  bridge endpoint breaks the bridge and releases its survivor.
- **FoeStats:** table values per level; `HasSkill1/2`, aggro/sight gates; evolution
  math (heal 30% of *new* max, +1 once, cap 8); spawn rule (level −2, min 1, no
  grandchildren).
- **Day-end pipeline:** step order (day → devour → CD tick → perish tick → NPC leave →
  stamina); held-only CD tick vs everywhere perish tick; 5-day NPC windows.
- **Modifier stack:** additive/multiplicative composition, source replace/clear,
  clamps, permanence-vs-buff separation.

### (4) In-game debug harnesses
Every feature ships with its harness the way the map did (dice/seed/Rest/Mist/views)
and combat did (`ShowDebugRanges`). Minimum for upcoming milestones: foe-level
override + sight-shape draw (v0.4.0), day/stamina/mission-state readout + "complete
goal" cheat (v0.5.0), give-item console (v0.6.0). Harnesses are debug-flagged, not
removed — they're also the reviewer's reproduction tools.

**The user manual for all debug tooling is `Docs/DEBUG_MODE.md`** (DBG toggle, SKIP,
key-5 log viewer + on-disk log, key-4 protagonist page, F9/R cheats, the
`DebugFoeLevel`/`DebugDay` exports). Keep it in sync when adding a harness.

### (5) Manual playtest protocol
Per milestone, a written checklist run on a **recorded seed**:
- The milestone's acceptance list (60-PRODUCTION §3) top-to-bottom.
- The **regression sweep** (§4 below).
- Log: seed, version, what felt wrong (feel notes are QA data too — see the shake
  caveat: "mathematically present, perceptually nothing").

## 2. Bug workflow (mandatory, from CLAUDE.md)

1. **Reproduce** — with a seed if run-dependent; note version.
2. **Fix** — smallest change that respects the architecture.
3. **Caveat** — append to `KNOWLEDGE.md` "Caveats / gotchas": *symptom → rule → why*,
   tagged with the version. This is non-negotiable; it is the studio's memory.
4. **Regression hook** — if the bug class is testable, add it to the unit tests or the
   regression sweep so it can't silently return (e.g. the function-node road-HUB rule:
   "don't remove this or the 'connected but walled-off' bug returns").
5. Version bump + changelog line.

## 3. What history says to watch (from KNOWLEDGE.md — the bug taxonomy)

- **Godot API drift:** no `Label2D`; Control `mouse_filter` STOP eats `_UnhandledInput`
  clicks; `_Ready` runs inside `AddChild`; fonts via `GetThemeDefaultFont()`.
- **Geometry:** hit tests must be radius-aware; `DrawColoredPolygon` needs convex;
  sprite size ≠ collision size — check both.
- **Feel wiring vs feel output:** an effect can be wired correctly and still invisible
  (trauma² shake). Verify magnitude, not call sites.
- **State bookkeeping:** coyote-time jump forfeiture needed three fields, not one flag.
  When a rule reads "X per Y unless Z", expect dedicated bookkeeping and test the
  unless-path.
- **Map rendering:** outer coast fills must be submitted as convex strips (never a concave
  polygon); altitude contours and tint are decorative only; roads remain the only route cue.
  Changes to `MapRenderModel.Build` need a coast/contour/road visual sweep across several seeds.

## 4. Regression sweep (run per milestone; grow this list)

Combat: move/jump/coyote (1-jump char can edge-jump, can't air-jump), platform
drop-through (hold = further), SoftVolume enter/swim/damping, 3-stage combo + reload,
Blush (+speed, 1.5× melee, tornado window 3 s, one per cast), seeds (per-wave 30/6
dedup, terrain kill), hazards (ticks bypass i-frames; tsunami one-shot respects them),
OnFire/Frozen decay to exactly 0, defense math, knockback on player AND foe, damage
numbers, shake visible, lives/respawn/win/lose/R-restart.
Map: seed reproduces; dice rerolls; one organic coast has no detached outer islets; the fixed
pentagonal VOID stays near 1/30 of the island; five variable-width rivers visibly divide exactly
five realm regions; each realm's nodes stay in its polygon and level ranks follow VOID distance;
every realm node reaches its 4-1 on roads that never cross a river, the VOID, or another road;
one of each realm's LV3→LV4 routes contains a Transportation Hub, every realm holds 3–5 hubs and
4–6 degree-two Event nodes, while Events split single ordinary roads; terrain labels match the
island altitude field; city-controlled field boundaries can be overlaid and clear height tint/contours
never imply a route; defeating the home LV4 awards TwistedReality, which builds from a completed
VOID-river peripheral city at no stamina cost and a 4-day cooldown, leaving an Eidolon Shelter
between endpoints that cannot bridge again while it survives;
stamina blocks at 0 without ending day; devour flickers then eats city-controlled fields in the
10/20/30/40/43/45 sequence, taking functional nodes only after a strict connected-city majority;
mist reveal rules; zone-6 one-way + `???` day + outer devour-all; both views + 3 cam modes;
wheel zoom; left-drag pan; right-drag player-pivot rotation; visited-grey re-walk.
Combat maps: matching world/hardship/goal/terrain selects the same map for the same seed and
node; no match falls back to legacy Arena; Character/Respawn/Enemy/Level Goal markers load;
claim/protect/destroy consume Level Goal markers while Slaughter ignores them; invalid spawn rows
are excluded before 4-second weighted foe selection.
Persistence: Play opens exactly three slots; empty slot starts a run and occupied slot restores
the same seed/day/node/map mutations/team/backpack/base stats. Verify `SaveGameService`
in-memory round trip; save at goal and a survivable timer failure, quit from map and battle,
then Continue. Map pause freezes movement; battle pause freezes spawns, mission timer, and
cooldowns. A battle Continue restarts its unfinished deterministic encounter with persisted
party state; Finish the Day persists the post-pipeline state; death/victory removes the slot.
Version stamp top-left matches `VERSION`.

## 5. Release gate (any commit that claims a milestone)

- [ ] Acceptance checklist passed on ≥2 seeds (one fresh, one archived "golden" seed)
- [ ] Regression sweep passed
- [ ] Unit tests green (if toolchain) / static checks clean (always)
- [ ] KNOWLEDGE.md updated for every bug fixed en route
- [ ] GDDs updated for every design decision made en route
- [ ] Version trio bumped; changelog in `Migration.md` §0
